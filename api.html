API Reference

Cube

Rubik's cube models can be created by instantiating new Cube objects. Cube objects store information about the state of the cube, and contain the methods that can be used to interact with the model.

The constructor takes two parameters:
  • frameID (Required): The id of the <div> element that will serve as the container for the Rubik's cube model.
  • options: An object containing additional options for the model.

Options

Property Default Type Description
mode "interactive" String Defines the appearence of the user interface section. One of "interactive" or "preset".
start [] Array of Strings The default starting orientation of the cube is blue in front and yellow on top.

A different starting orientation can be specified by setting startto be an array of moves (specified using standard cube notation). These moves will be applied to the default starting orientation to obtain a new starting orientation.
sequence [] Array of Strings Defines the pre-set sequence of moves that the user can cycle through (in "preset" mode).
hideInterface false Boolean Defines whether the default user interface is shown.
onClick undefined Function Defines a callback function to be invoked whenever the user clicks a button in the user interface.

Methods

Name Description
R, RPrime, L, LPrime, U, UPrime, D, DPrime, F, FPrime, B, BPrime, X, XPrime, Y, YPrime, Z, ZPrime These methods modify the cube by performing a move (specified using standard cube notation).
isSolved Return whether the cube is in a solved state.
scramble Scramble the cube using a sequence of random moves.
next Apply the next move in the pre-set sequence of moves (if such a move exists).
previous Reverse the previous move in the pre-set sequence of moves (if such a move exists).
